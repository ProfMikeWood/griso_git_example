# A Git/Hub Primer
The is an example repository to show some Git/Hub tools during the GRISO workshop.

## What is a GitHub repository used for?
GitHub repositories can be used for a variety of purposes! Here are a few examples of public repositories:
- Example of modeling source code: [MITgcm](https://github.com/MITgcm/MITgcm)
- Example of a software package: [diagnostics_vec](https://github.com/mhwood/diagnostics_vec) (by Mike Wood)
- Example of a project to support a publication: [sermilik_gridded_hydrography](https://github.com/a1roth/sermilik_gridded_hydrography) (by Aurora Roth)
- Example of notebooks to generate a "book": [ocean_modeling_book](https://github.com/ProfMikeWood/ocean_modeling_book) (by Mike Wood, see book [HERE](https://profmikewood.github.io/ocean_modeling_book/intro.html))

## How do I get code from with someone's GitHub repository?
To get code from someone's repository, there are a few options:
1. Download a zipped file of the repository.
   
   You can do this by clicking on the big green Code button available on the repository page. Here, you will see an option for "Download ZIP"
   
2. "Clone" the code using command line tools

   On your command line (e.g. Terminal on Mac, Git Bash on Windows), you can use the command `git pull git@github.com:repository_name.git`. This option is advantangeous when the author of the original code makes regular updates to the code base because you can repeat the `git pull` command to update your version of the code with your local version.
   
3. "Fork" the code on GitHub

   This option is similar to the previous option but it will create a copy of the code base in a new repository on your account. This option is typically undertaken when you plan to make updates to the code base but need a separate working version for testing your changes.
